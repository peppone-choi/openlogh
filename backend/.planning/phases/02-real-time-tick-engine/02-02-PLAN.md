<feature>
  <name>TickEngine - Real-time game clock with month boundary and CP regen gating</name>
  <files>
    backend/game-app/src/main/kotlin/com/openlogh/engine/TickEngine.kt
    backend/game-app/src/test/kotlin/com/openlogh/engine/TickEngineTest.kt
  </files>
  <behavior>
    - processTick increments tickCount by 1 and gameTimeSec by GAME_SECONDS_PER_TICK (24)
    - After 1 tick: tickCount=1, gameTimeSec=24
    - After 100 ticks: tickCount=100, gameTimeSec=2400
    - processCompletedCommands is called every tick
    - regenerateCommandPoints is called only when tickCount % 300 == 0 (not on tick 0)
    - At tick 300: CP regen fires. At tick 299: does not fire. At tick 600: fires again.
    - Monthly pipeline fires when gameTimeSec crosses GAME_SECONDS_PER_MONTH (2,592,000)
    - On month crossing: gameTimeSec resets by subtracting GAME_SECONDS_PER_MONTH, currentMonth advances
    - If currentMonth goes past 12, currentYear increments and currentMonth resets to 1
    - Monthly pipeline does NOT fire on normal ticks (tickCount=1..107999 with no boundary crossing)
    - tickCount never resets -- it accumulates across months
  </behavior>
  <implementation>
    1. Create TickEngine as @Service with dependencies: RealtimeService, TurnService (for monthly pipeline), SessionStateRepository, GameEventService.

    2. `processTick(world: SessionState)` method:
       a. world.tickCount++
       b. world.gameTimeSec += GAME_SECONDS_PER_TICK
       c. Call realtimeService.processCompletedCommands(world)
       d. If world.tickCount % CP_REGEN_INTERVAL_TICKS == 0L: call realtimeService.regenerateCommandPoints(world)
       e. If world.gameTimeSec >= GAME_SECONDS_PER_MONTH:
          - world.gameTimeSec -= GAME_SECONDS_PER_MONTH
          - advanceMonth(world) -- increment currentMonth, handle year rollover
          - Call turnService.runMonthlyPipeline(world) (extract monthly pipeline from TurnService.processWorld, excluding command execution)
       f. Save world via sessionStateRepository.save(world)

    3. `advanceMonth(world)`: increment currentMonth. If currentMonth > 12, set currentMonth=1 and increment currentYear. This mirrors what TurnService already does internally.

    4. Update TurnDaemon:
       - Change @Scheduled annotation: fixedDelayString -> fixedRateString, value from "${app.turn.interval-ms:5000}" to "${app.turn.tick-ms:1000}"
       - In tick() for realtime worlds: replace direct realtimeService calls with tickEngine.processTick(world)
       - Keep the non-realtime (turn-based) path unchanged for backward compatibility

    5. Update RealtimeService.regenerateCommandPoints: NO changes needed to the method itself. The gating is handled by TickEngine's modular check. The method already does the right thing (regen CP for all officers in the world).

    Note on TurnService monthly pipeline extraction: If TurnService.processWorld is too entangled to cleanly extract a "monthly pipeline only" method, create a thin wrapper in TickEngine that calls the existing turn processing steps (economy, diplomacy, maintenance, etc.) but skips command execution. Research shows TurnService already has separate method calls for each pipeline stage -- call those directly.
  </implementation>
</feature>
