    Add a new method to GameEventService after the existing broadcastTurnAdvance method:

    ```kotlin
    fun broadcastTickState(world: SessionState) {
        messagingTemplate.convertAndSend(
            "/topic/world/${world.id}/tick",
            mapOf(
                "type" to "tick_state",
                "gameTimeSec" to world.gameTimeSec,
                "tickCount" to world.tickCount,
                "year" to world.currentYear.toInt(),
                "month" to world.currentMonth.toInt(),
                "gameDayOfMonth" to (world.gameTimeSec / GameTimeConstants.GAME_SECONDS_PER_DAY + 1),
                "gameHour" to ((world.gameTimeSec % GameTimeConstants.GAME_SECONDS_PER_DAY) / 3600),
                "serverTimestamp" to System.currentTimeMillis(),
            )
        )
    }
    ```

    Add the necessary import at the top of the file:
    ```kotlin
    import com.openlogh.engine.GameTimeConstants
    import com.openlogh.entity.SessionState
    ```

    The SessionState import may already exist. Only add if missing. The method sends to `/topic/world/{id}/tick` which is a new WebSocket topic. Clients subscribe to this to get game clock updates.

    The payload includes:
    - gameTimeSec: raw game-seconds within current month (for precise calculations)
    - tickCount: absolute tick count (for client-side interpolation reference)
    - year/month: current calendar values
    - gameDayOfMonth: derived day (1-30) for display
    - gameHour: derived hour (0-23) for display
    - serverTimestamp: wall-clock ms for client drift correction
