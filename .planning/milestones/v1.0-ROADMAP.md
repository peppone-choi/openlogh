**Status:** SHIPPED 2026-04-03

## Milestone Summary

**Key Decisions:**
- Code-only parity verification (PHP source is sole authority, not docs)
- Golden-value test approach: fixed-seed RNG output captured and locked as expected values
- LiteHashDRBG for deterministic RNG with PHP SHA-512 stream parity
- kotlin.math.round for PHP rounding parity (banker's rounding .5 divergence documented)
- Nation level thresholds extended from PHP 8-level to opensamguk 10-level (intentional)
- RNG seed divergence documented (Kotlin "disaster" vs PHP "disater" typo -- not fixed to avoid world data impact)
- Scenario general stats use 삼국지14 values (intentional divergence from legacy 3-stat)

**Issues Resolved:**
- java.util.Random non-determinism replaced with LiteHashDRBG
- Short field overflow causing silent data corruption
- SpecialModifiers actionCode mismatch for domestic commands
- NPC diplomacy AI "completely different" from legacy -- rewritten to match
- Missing WarUnitTrigger hooks in BattleEngine
- 4 turn engine stub methods left unimplemented

**Issues Deferred:**
- War term casualty extension (marked @Disabled as potential parity gap)
- AuthService deduplication between gateway-app and game-app

**Technical Debt Incurred:**
- Che*Trigger no-op objects kept in BattleTriggerRegistry to avoid breaking presence checks
- NationAI.adjustTaxAndBill runtime-active but non-PHP-matching (GeneralAI rate choosers are the PHP-matching path)
