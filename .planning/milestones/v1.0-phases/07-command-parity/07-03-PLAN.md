## 국가 커맨드 검증 방식 (Pitfall 4 대응)

국가 커맨드는 장수 커맨드와 달리 run()에서 직접 entity를 수정한다 (message JSON delta가 아님).
따라서 검증 방식이 다르다:
```kotlin
// 장수 커맨드: result.message JSON delta 검증
val result = cmd.run(rng)
assertEquals(expectedDelta, mapper.readTree(result.message))

// 국가 커맨드: run() 전후 entity 상태 diff 검증
val beforeGold = nation.gold
val result = cmd.run(rng)
assertEquals(expectedAfterGold, nation.gold)  // 직접 entity 필드 비교
assertEquals(expectedGoldDelta, nation.gold - beforeGold)
```
