# Phase 6 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Classification: Backend Only

**This phase has NO frontend UI deliverables.**

Phase 6 (Economy Parity) verifies that all economic calculations in the Kotlin backend produce identical values to the legacy PHP implementation. The scope is:

- ECON-01: Tax collection formula verification (golden value tests)
- ECON-02: Trade income formula verification (golden value tests)
- ECON-03: Supply/food consumption formula verification (golden value tests)
- ECON-04: Population growth/decline formula verification (golden value tests)
- ECON-05: City development formula verification (golden value tests)
- ECON-06: Semi-annual salary distribution verification (golden value tests)

All work is in `backend/game-app/src/test/kotlin/` and `backend/game-app/src/main/kotlin/`. No frontend components, pages, or styles are created or modified.

**Source:** CONTEXT.md D-01 through D-04, RESEARCH.md Phase Requirements table, ROADMAP.md Phase 6 description.
