### Task 3: TDD — 24-tick drain regression invariant

Create `EconomyPipelineRegressionTest.kt`:
```kotlin
@Test fun `running 24 monthly ticks on empty NPC world does not drain faction funds by more than 10 percent`() {
    val world = buildMinimalNpcWorld(factions = 2, planetsPerFaction = 3, officersPerFaction = 5)
    val initialTotalFunds = world.factions.sumOf { it.funds }
    repeat(24) { turnService.processMonth(world) }
    val finalTotalFunds = world.factions.sumOf { it.funds }
    val drainPercent = (initialTotalFunds - finalTotalFunds).toDouble() / initialTotalFunds
    assertTrue(drainPercent < 0.10, "Drain: ${drainPercent * 100}% — invariant is <10%")
}
```

Additional scenarios:
1. 24-tick with war state → war income keeps funds positive or within 5% drain
2. Per-resource isolation: `supplies` unchanged during pure-funds month
3. Salary outlay annual check: officer.funds increases by ~getBillFromDedication(ded) × taxRate/100
4. Phase 14 regression: tactical battle tests still pass after pipeline wire-up

Commit RED first (may initially fail due to integration bugs), iterate until GREEN.
