# Plan 23-01: Gin7EconomyService.processIncome per-resource body

## Objective

Implement real per-resource income body in `Gin7EconomyService.processIncome(world, resource)` ported from upstream `com.opensam.engine.EconomyService.processIncome`. The method must process ONLY the specified resource (funds OR supplies, not both) to match the legacy month 1 / month 7 schedule.

## Legacy reference extraction

Read the upstream method body via:
```bash
git show a7a19cc3 -- backend/game-app/src/main/kotlin/com/opensam/engine/EconomyService.kt | awk '/fun processIncome/,/^\s*\}/'
```

The legacy method computes:
1. For each faction: income = sum over supplied planets of (planet.commerce × taxRate / 100)
2. Applies `funds += income` OR `supplies += (production-based supplies calculation)`
3. Triggers approval adjustment based on tax rate
4. Saves faction + planet state via ports

Note: LOGH's `Gin7EconomyService.processMonthly` already has tax collection for month 1/4/7/10. This plan ADDS the per-resource `processIncome(world, resource)` public method as a separate entry point that the scheduled events (month 1 = "gold" → funds, month 7 = "rice" → supplies) call. The `processMonthly` tax collection path can call `processIncome(world, "gold")` internally to DRY up.

## Tasks

### Task 1: TDD RED — per-resource isolation contract

Create `Gin7ProcessIncomeTest.kt` with:
1. `processIncome(world, "gold")` modifies `faction.funds` but NOT `faction.supplies`
2. `processIncome(world, "rice")` modifies `faction.supplies` but NOT `faction.funds`
3. Invalid resource throws `IllegalArgumentException`
4. Empty world (no factions) runs without error
5. Isolated planets (`supplyState == 0`) are excluded from the per-resource calculation

Commit RED with `--no-verify`.

### Task 2: GREEN — port the method

1. Add `fun processIncome(world: SessionState, resource: String)` to `Gin7EconomyService`
2. `require(resource == "gold" || resource == "rice")`
3. Branch on resource literal; call internal `processGoldIncome(faction, planets)` / `processSuppliesIncome(faction, planets)` helpers
4. For `"gold"`: sum commerce × taxRate / 100 over supplied planets → funds
5. For `"rice"`: sum production × supplyRatio over supplied planets → supplies (port the legacy production-based rice formula)
6. Persist via `factionRepository.saveAll`

Verify all 5 tests pass.

Commit GREEN with `--no-verify`.
