# Phase 1 — UI Design Contract

> Visual and interaction contract for the session foundation frontend work. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property          | Value                                                                                             |
| ----------------- | ------------------------------------------------------------------------------------------------- |
| Tool              | shadcn (new-york style)                                                                           |
| Preset            | new-york, neutral base, CSS variables, RSC enabled                                                |
| Component library | Radix UI (via shadcn)                                                                             |
| Icon library      | Lucide React 0.564.0                                                                              |

**Source:** `frontend/components.json`, `frontend/src/app/globals.css`

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage                                                                   |
| ----- | ----- | ----------------------------------------------------------------------- |
| xs    | 4px   | Icon gaps, inline badge padding, table cell vertical padding            |
| sm    | 8px   | Compact element spacing, card content gaps, button gaps                 |
| md    | 14px  | Card content padding, card header horizontal padding (existing pattern) |
| lg    | 24px  | Section spacing between cards (`space-y-6` = 24px)                      |
| xl    | 32px  | Layout gaps between major panels                                        |
| 2xl   | 48px  | Empty state vertical padding (`py-12` = 48px)                           |
| 3xl   | 64px  | Page-level spacing (not used in Phase 1)                                |

Exceptions:

- Card header min-height: 38px (existing pattern from globals.css `[data-slot='card-header']`)
- Mobile touch targets: 44px min-height on buttons and card headers at viewport <= 768px (existing responsive rule)
- Card content padding uses 14px (not 16px) to match established `[data-slot='card-content']` pattern

**Source:** Extracted from `globals.css` existing `[data-slot]` rules and lobby page Tailwind classes.

## Typography

| Role            | Size             | Weight         | Line Height | Font                                        |
| --------------- | ---------------- | -------------- | ----------- | ------------------------------------------- |
| Body            | 14px             | 400 (regular)  | 1.3         | `--font-legacy` (Pretendard stack)          |
| Label / Caption | 11px-12px        | 500 (medium)   | 1.15        | `--font-legacy`                             |
| Card Title      | 14px             | 600 (semibold) | 1.4         | `--font-legacy`                             |
| Section Heading | 18px (`text-lg`) | 600 (semibold) | 1.4         | `--font-dunggeunmo` (via `game-font` class) |

Supplementary sizes used in existing components (not new declarations):

- 10px: Micro badges (`text-[10px]`)
- 13px: Button text, table cells, table body
- 20px (`text-xl`): Officer name display in detail panel

**Source:** `globals.css` base `body` rule (14px/1.3), `[data-slot='card-title']` (14px/600/1.4), `[data-slot='badge']` (11px/500), lobby page heading pattern (`game-font text-lg font-semibold`).

## Color

All colors are pre-defined in `globals.css` `:root` and `.dark` blocks. Phase 1 does NOT introduce new colors.

| Role            | CSS Variable    | Hex Value | Usage                                                                                        |
| --------------- | --------------- | --------- | -------------------------------------------------------------------------------------------- |
| Dominant (60%)  | `--background`  | `#0a0a1a` | Page background (space-dark), body bg `#050510` with star-field gradient                     |
| Secondary (30%) | `--card`        | `#1a1a2e` | Card surfaces, input backgrounds, popover backgrounds                                        |
| Accent (10%)    | `--primary`     | `#c9a84c` | Primary CTA buttons, active tab indicator, selected card border, ring/focus, scrollbar thumb |
| Destructive     | `--destructive` | `#c41e3a` | Error state icon tint, destructive button variant, form validation errors                    |

### Faction Colors (Phase 1 specific)

| Faction           | CSS Variable             | Hex       | Usage in Phase 1                                           |
| ----------------- | ------------------------ | --------- | ---------------------------------------------------------- |
| Empire (Gold)     | `--empire-gold`          | `#c9a84c` | Empire faction picker option, session list faction count   |
| Alliance (Blue)   | `--alliance-blue`        | `#1e4a8a` | Alliance faction picker option, session list faction count |
| Alliance (Bright) | `--alliance-blue-bright` | `#4a9eff` | Alliance faction badge dot color                           |

Accent reserved for:

- Primary CTA buttons (`[data-variant='default']` button)
- Active tab triggers (`[data-state='active']`)
- Selected card border highlight (`border-primary`)
- Focus ring (`--ring`)
- Section headings with `game-font` class (gold-tinted backgrounds via gradient)
- Scrollbar thumb

**Source:** `globals.css` `:root` block, faction color system classes.

## Interaction Contract

### Session List

| Interaction        | Behavior                                                         |
| ------------------ | ---------------------------------------------------------------- |
| Click session card | Selects session, loads detail in right panel (existing pattern)  |
| Selected state     | Card border changes to `border-primary` with `bg-primary/5` tint |
| Hover state        | Card border `hover:border-primary/50` (existing pattern)         |
| Loading            | `LoadingState` with spinner + "로딩 중..."                       |
| Empty              | `EmptyState` with globe icon + "생성된 세션이 없습니다"          |
| Error              | `ErrorState` with retry button                                   |

### Faction Picker (Join Page Enhancement)

| Interaction           | Behavior                                                                                                       |
| --------------------- | -------------------------------------------------------------------------------------------------------------- |
| Empire option         | Card with empire-gold border accent, faction icon, current player count, ratio bar fill                        |
| Alliance option       | Card with alliance-blue border accent, faction icon, current player count, ratio bar fill                      |
| Selected state        | Faction card gets solid faction-color border + subtle background tint (`bg-[faction]/5`)                       |
| Blocked (cap reached) | Card gets `opacity-50 cursor-not-allowed`, blocked message appears below in `text-muted-foreground`            |
| Ratio bar             | Horizontal bar: empire-gold left segment + alliance-blue right segment, 60% cap marker as dashed vertical line |

### Session Creation

| Interaction      | Behavior                                                                                                 |
| ---------------- | -------------------------------------------------------------------------------------------------------- |
| Scenario select  | Dropdown (`Select` component) listing available scenarios by name                                        |
| Create button    | `Button` default variant, "세션 생성" label, disabled while submitting                                   |
| Submitting state | Button shows `Loader2` spinner + "생성중..." label, pointer-events disabled                              |
| Success          | Redirect to lobby with new session selected, toast "세션이 생성되었습니다"                               |
| Error            | Inline error message in `bg-destructive/20 text-destructive` container (existing pattern from join page) |

### Destructive Actions

Phase 1 has no destructive actions. Session deletion is admin-only and not in scope.

## Phase 1 UI Scope Summary

Phase 1 is **backend-heavy**. The frontend changes are limited to:

1. **Enhance session list** (D-04): Show Empire/Alliance player counts separately in each session card
2. **Add faction picker** (SESS-03 + D-01/D-02): Replace generic nation dropdown with binary Empire/Alliance picker with ratio enforcement display
3. **Add session creation** (SESS-01 + D-03): Scenario-only creation flow (either new page or dialog)
4. **Update lobby header** text from "오픈삼국" to "오픈 은하영웅전설"

Requirements HARD-01, HARD-02, SMGT-01, SESS-06, SESS-07 are **backend-only** with no frontend UI changes.
