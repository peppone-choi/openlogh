# Plan 22-02: OfficerAI.doDonate — probability gate on excess branches

## Objective

Port the upstream GeneralAI.doDonate fix from commit `a7a19cc3` into LOGH's `OfficerAI.doDonate`. Currently the "excess gold/rice" branches donate unconditionally, causing officers to burn through personal stockpiles every turn.

## Bug reproduction

In LOGH `OfficerAI.doDonate` (line 2375), the pattern likely mirrors upstream's bug:

```kotlin
// Excess gold even if nation doesn't need it
if (!donateGold && general.gold > reqGold * 5 && general.gold > 5000) {
    donateGold = true  // ← unconditional
}
```

This makes every NPC officer with > 5000 gold donate on every tick, draining personal funds at ~11.5%/year (1만 초과 구간).

## Upstream fix summary

Wrap each unconditional branch in the same probability check used in the "nation needs" branch:

```kotlin
if (!donateGold && general.gold > reqGold * 5 && general.gold > 5000) {
    if (reqGold > 0 && rng.nextDouble() < (general.gold.toDouble() / reqGold - 0.5)) {
        donateGold = true
    }
}
```

Also add a `reqGold > 0 &&` guard to the nation-needs branch to prevent divide-by-zero.

Same treatment for the rice branches.
