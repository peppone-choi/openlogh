# Plan 22-01: FactionAI.adjustTaxAndBill ŌĆö legacy-correct bill formula

## Objective

Port the upstream NationAI.adjustTaxAndBill fix from commit `a7a19cc3` into LOGH's `FactionAI.adjustTaxAndBill`. The current formula underestimates actual NPC faction salary outlay by ~4x, causing treasury drain.

## Bug reproduction

Current LOGH (`FactionAI.kt:327`):
```kotlin
val totalBill = (nationGenerals.size + nationCities.size) * nation.taxRate.toInt().coerceAtLeast(0)
```

This conflates "number of officers+planets" with "bill amount". The result is compared against `nation.funds < totalBill * 3` to decide whether to lower the tax rate ŌĆö but because `totalBill` is so wrong, the rate lowering fires incorrectly and the faction's funds drain.

## Upstream fix summary

1. Compute `baseOutcome = sum of getBill(dedication) for each active officer`
2. `totalBill = baseOutcome * taxRate / 100`
3. Clamp minimum `taxRate` to 20 (not 0) so it can never drop to zero
4. Add `getBillFromDedication(dedication)` helper matching PHP `getBill()`:
   - `getBill(ded) = getDedLevel(ded) * 200 + 400`
   - `getDedLevel(ded) = clamp(ceil(sqrt(ded) / 10), 0, 30)`

## Tasks

### Task 1: TDD RED ŌĆö failing test for the bug

Create `backend/game-app/src/test/kotlin/com/openlogh/engine/ai/FactionAIBillFormulaTest.kt` with 4 cases:

1. `getBillFromDedication(0) == 400` ŌĆö dedLevel 0 ŌåÆ 0*200+400
2. `getBillFromDedication(100) == 600` ŌĆö sqrt(100)/10=1 ŌåÆ ceil=1 ŌåÆ 1*200+400
3. `getBillFromDedication(10000) == 4400` ŌĆö sqrt(10000)/10=10 ŌåÆ 10*200+400
4. Anchor test: faction with 10 officers (avg ded 400) at taxRate=100 should have `totalBill == sum(getBill(400))`, NOT `10 * 100`
