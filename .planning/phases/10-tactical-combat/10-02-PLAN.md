# Plan 10-02: Tactical Battle Engine & Trigger Service

## Objective
Implement the core real-time battle engine: BattleTriggerService (detects battles), TacticalBattleEngine (processes battle ticks with energy, formations, fortress guns, damage resolution).

## Context
- @backend/game-app/src/main/kotlin/com/openlogh/engine/war/BattleEngine.kt — existing legacy battle engine
- @backend/game-app/src/main/kotlin/com/openlogh/engine/war/WarUnit.kt — existing war unit abstraction
- @backend/game-app/src/main/kotlin/com/openlogh/engine/war/WarFormula.kt — tech/dex formulas
- @backend/game-app/src/main/kotlin/com/openlogh/model/FortressType.kt — fortress gun stats
- @backend/game-app/src/main/kotlin/com/openlogh/service/GameEventService.kt — WebSocket broadcasting

## Tasks

### Task 1: Create TacticalBattleEngine
type="auto"

Create `com.openlogh.engine.tactical.TacticalBattleEngine`:
1. `TacticalUnit` data class — runtime unit for battle (fleetId, officerId, factionId, ships, hp, maxHp, position, energy allocation, formation, velocity, commandRange, stats)
2. `TacticalBattleState` data class — full battle state (units list, tick count, fortress gun state, battle bounds)
3. `processTick(state: TacticalBattleState): TacticalBattleState` — single tick logic:
   - Movement: units move based on ENGINE energy and formation speed modifier
   - Command range: expands over time based on officer's command stat, resets to 0 on new order
   - Weapon fire: BEAM (mid-close range), GUN (mid-close range) — damage based on energy allocation
   - Shield absorption: SHIELD energy reduces incoming damage
   - SENSOR: affects accuracy/detection range
   - WARP: emergency retreat capability
   - Morale/training effects on combat performance
4. `checkBattleEnd(state: TacticalBattleState): BattleOutcome?` — check victory conditions
5. `applyFormationModifiers(unit: TacticalUnit)` — apply formation bonuses

Done when: Engine compiles, tick processing covers energy/formation/damage/movement.

### Task 2: Create BattleTriggerService and FortressGunService
type="auto"

Create:
1. `com.openlogh.engine.tactical.BattleTriggerService` (@Service):
   - `checkForBattles(sessionId: Long)` — find opposing fleets at same star system or route
   - `createBattle(sessionId, starSystemId, attackerFleets, defenderFleets): TacticalBattle` — initialize battle instance
   - `buildInitialState(battle: TacticalBattle): TacticalBattleState` — create initial unit positions from fleet data
2. `com.openlogh.engine.tactical.FortressGunService`:
   - `canFire(starSystem: StarSystem, tickCount: Int): Boolean` — cooldown check
   - `calculateLineOfFire(gunPosition, targetPosition, allUnits): List<TacticalUnit>` — line-of-fire hits ALL units in path (including friendlies per gin7)
   - `fireFortressGun(starSystem, state): TacticalBattleState` — apply fortress gun damage

Done when: Both services compile, fortress gun line-of-fire calculation handles friendly fire.
