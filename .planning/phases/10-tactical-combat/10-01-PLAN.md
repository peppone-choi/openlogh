# Plan 10-01: Tactical Battle Models & DB Migration

## Objective
Create the data model foundation for real-time tactical combat: TacticalBattle entity, EnergyAllocation value object, Formation enum, and Flyway migration V37.

## Context
- @backend/game-app/src/main/kotlin/com/openlogh/entity/Fleet.kt — fleet structure
- @backend/game-app/src/main/kotlin/com/openlogh/entity/StarSystem.kt — star system with fortress guns
- @backend/game-app/src/main/kotlin/com/openlogh/model/FortressType.kt — fortress gun stats
- @backend/game-app/src/main/kotlin/com/openlogh/model/UnitType.kt — unit types

## Tasks

### Task 1: Create tactical combat model enums and value objects
type="auto"

Create:
1. `EnergyAllocation` data class in `com.openlogh.model` — 6 channels (BEAM, GUN, SHIELD, ENGINE, WARP, SENSOR) with total energy constraint (sum = 100)
2. `Formation` enum in `com.openlogh.model` — 4 types (WEDGE, BY_CLASS, MIXED, THREE_COLUMN) with attack/defense/speed modifiers
3. `BattlePhase` enum in `com.openlogh.model` — battle lifecycle states (PREPARING, ACTIVE, PAUSED, ENDED)
4. `TacticalUnitState` data class — per-unit state in battle (position x/y, hp, shields, energy allocation, formation, velocity)

Done when: All model classes compile and have correct modifier values per gin7 manual.

### Task 2: Create TacticalBattle entity and Flyway V37 migration
type="auto"

Create:
1. `TacticalBattle` JPA entity in `com.openlogh.entity`:
   - id, sessionId, starSystemId (where battle occurs)
   - attackerFactionId, defenderFactionId
   - phase (BattlePhase as VARCHAR)
   - startedAt, endedAt (OffsetDateTime)
   - participants (JSONB — list of fleet IDs per side)
   - battleState (JSONB — current tactical state snapshot)
   - result (VARCHAR nullable — "attacker_win", "defender_win", "draw")
   - tickCount (Int — battle ticks elapsed)
2. `TacticalBattleRepository` in `com.openlogh.repository`
3. Flyway `V37__tactical_battle.sql` migration

Done when: Migration SQL is valid, entity class compiles, repository interface defined.

## Verification
- All new Kotlin files compile: `cd backend && ./gradlew :game-app:compileKotlin`

## Success Criteria
- EnergyAllocation enforces sum=100 constraint
- Formation enum has 4 types with distinct modifiers
- TacticalBattle entity maps to tactical_battle table
- V37 migration creates the table with correct columns
