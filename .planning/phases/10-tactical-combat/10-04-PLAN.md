# Plan 10-04: Frontend Tactical Battle UI

## Objective
Build the frontend tactical battle view: 2D battle map showing fleet positions, energy allocation panel, formation selector, battle status display, and real-time WebSocket updates.

## Context
- @frontend/src/lib/websocket.ts — existing WebSocket client
- @frontend/src/app/(game)/battle/page.tsx — existing battle page (military overview)
- @frontend/src/stores/ — Zustand store pattern
- @frontend/src/components/ui/8bit/ — 8-bit UI component library

## Tasks

### Task 1: Create tactical battle types, API client, and Zustand store
type="auto"

Create:
1. `frontend/src/types/tactical.ts` — TypeScript types:
   - TacticalBattle, TacticalUnit, EnergyAllocation, Formation, BattlePhase, BattleCommand
2. `frontend/src/lib/tacticalApi.ts` — API client:
   - getActiveBattles(sessionId), getBattleState(sessionId, battleId)
   - sendBattleCommand(sessionId, command) via WebSocket
3. `frontend/src/stores/tacticalStore.ts` — Zustand store:
   - currentBattle, units, myUnit, energyAllocation, formation
   - subscribeToBattle(sessionId, battleId) — WebSocket subscription
   - setEnergy(allocation), setFormation(formation), retreat()

Done when: Types/API/store compile with `pnpm build`.

### Task 2: Create tactical battle UI components
type="auto"

Create:
1. `frontend/src/components/tactical/BattleMap.tsx` — 2D canvas (React Konva) showing:
   - Unit positions as ship icons with faction colors
   - Command range circles
   - Fortress gun range indicators
   - Damage numbers floating up
2. `frontend/src/components/tactical/EnergyPanel.tsx` — 6-slider energy allocation:
   - BEAM/GUN/SHIELD/ENGINE/WARP/SENSOR sliders
   - Total must equal 100, auto-adjust when moving one slider
   - Visual feedback for current allocation
3. `frontend/src/components/tactical/FormationSelector.tsx` — 4 formation buttons:
   - Wedge/By-class/Mixed/Three-column with modifier tooltips
4. `frontend/src/components/tactical/BattleStatus.tsx` — battle info:
   - Tick count, elapsed time, side HP bars, unit count per side
5. `frontend/src/app/(game)/tactical/page.tsx` — tactical battle page
   - Composes BattleMap + EnergyPanel + FormationSelector + BattleStatus

Done when: All components render without errors, page route works.

## Verification
- Frontend builds: `cd frontend && pnpm build`

## Success Criteria
- Tactical battle page shows 2D battle map with unit positions
- Energy panel allows 6-channel allocation with sum=100 constraint
- Formation selector shows 4 options with modifier descriptions
- Battle status shows real-time HP bars and tick counter
- WebSocket subscription receives and displays battle updates
