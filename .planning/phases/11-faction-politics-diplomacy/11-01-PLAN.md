# Plan 11-01: Empire Politics - Nobility & Coup System

## Objective
Create the Empire's autocratic governance model: 5-level nobility hierarchy, coup state machine (PLANNING->ACTIVE->SUCCESS/FAILED/ABORTED), and Flyway V38 migration for political tables.

## Context
- @backend/game-app/src/main/kotlin/com/openlogh/entity/Faction.kt -- faction entity with meta JSONB
- @backend/game-app/src/main/kotlin/com/openlogh/entity/Officer.kt -- officer entity with positionCards, meta JSONB
- @backend/game-app/src/main/kotlin/com/openlogh/model/PositionCard.kt -- EMPEROR, SUPREME_COMMANDER cards
- @backend/game-app/src/main/kotlin/com/openlogh/service/ScenarioEventService.kt -- coup/rebellion events already defined
- @docs/reference/scenarios_detail.md -- coup mechanics (Greenhill coup at politics 8000, Braunschweig rebellion)

## Tasks

### Task 1: Create Empire political model enums and entities
type="auto"

Create:
1. `NobilityRank` enum in `com.openlogh.model` -- 5 peerage levels:
   - COMMONER (평민, no political bonus)
   - BARON (남작, +5% politics influence)
   - COUNT (백작, +10% politics influence)
   - MARQUIS (후작, +15% politics influence)
   - DUKE (공작, +20% politics influence)
   Each has: code, nameKo, nameEn, politicsBonus (Float), minRank (Int)

2. `CoupPhase` enum in `com.openlogh.model` -- coup lifecycle:
   - PLANNING (비밀 모의 중)
   - ACTIVE (쿠데타 진행 중)
   - SUCCESS (성공)
   - FAILED (실패)
   - ABORTED (중단)

3. `CoupEvent` JPA entity in `com.openlogh.entity`:
   - id, sessionId, factionId
   - leaderId (Long -- officer who initiated)
   - phase (CoupPhase as VARCHAR)
   - supporterIds (JSONB -- list of officer IDs supporting the coup)
   - targetSovereignId (Long -- current sovereign being overthrown)
   - politicalPower (Int -- accumulated political machination points)
   - threshold (Int -- points needed to trigger, default 8000 per gin7)
   - startedAt, resolvedAt (OffsetDateTime)
   - result (VARCHAR nullable)
   - meta (JSONB)

4. `CoupEventRepository` in `com.openlogh.repository`

5. `NobilityRecord` data stored in Officer.meta as `nobility_rank` key (String value matching NobilityRank.code). No separate entity needed -- uses existing JSONB meta field.

Done when: All model classes and entity compile. NobilityRank has 5 levels. CoupPhase has 5 states.

### Task 2: Create Flyway V38 migration for faction politics tables
type="auto"

Create `V38__faction_politics.sql`:
1. `coup_event` table:
   - id BIGSERIAL PRIMARY KEY
   - session_id BIGINT NOT NULL REFERENCES session_state(id)
   - faction_id BIGINT NOT NULL REFERENCES faction(id)
   - leader_id BIGINT NOT NULL REFERENCES officer(id)
   - phase VARCHAR(20) NOT NULL DEFAULT 'PLANNING'
   - supporter_ids JSONB NOT NULL DEFAULT '[]'
   - target_sovereign_id BIGINT NOT NULL
   - political_power INT NOT NULL DEFAULT 0
   - threshold INT NOT NULL DEFAULT 8000
   - started_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
   - resolved_at TIMESTAMPTZ
   - result VARCHAR(20)
   - meta JSONB NOT NULL DEFAULT '{}'
   - INDEX on (session_id, faction_id, phase)

2. `council_seat` table (for Alliance, used in Plan 11-02):
   - id BIGSERIAL PRIMARY KEY
   - session_id BIGINT NOT NULL REFERENCES session_state(id)
   - faction_id BIGINT NOT NULL REFERENCES faction(id)
   - seat_code VARCHAR(50) NOT NULL
   - officer_id BIGINT REFERENCES officer(id)
   - elected_at TIMESTAMPTZ
   - term_end_at TIMESTAMPTZ
   - votes_received INT NOT NULL DEFAULT 0
   - meta JSONB NOT NULL DEFAULT '{}'
   - UNIQUE(session_id, faction_id, seat_code)

3. `fezzan_loan` table (for Fezzan, used in Plan 11-03):
   - id BIGSERIAL PRIMARY KEY
   - session_id BIGINT NOT NULL REFERENCES session_state(id)
   - borrower_faction_id BIGINT NOT NULL REFERENCES faction(id)
   - principal INT NOT NULL
   - interest_rate REAL NOT NULL DEFAULT 0.05
   - remaining_debt INT NOT NULL
   - issued_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
   - due_at TIMESTAMPTZ NOT NULL
   - repaid_at TIMESTAMPTZ
   - is_defaulted BOOLEAN NOT NULL DEFAULT FALSE
   - meta JSONB NOT NULL DEFAULT '{}'
   - INDEX on (session_id, borrower_faction_id)

4. `election` table (for Alliance, used in Plan 11-02):
   - id BIGSERIAL PRIMARY KEY
   - session_id BIGINT NOT NULL REFERENCES session_state(id)
   - faction_id BIGINT NOT NULL REFERENCES faction(id)
   - election_type VARCHAR(30) NOT NULL
   - started_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
   - ended_at TIMESTAMPTZ
   - candidates JSONB NOT NULL DEFAULT '[]'
   - votes JSONB NOT NULL DEFAULT '{}'
   - winner_officer_id BIGINT REFERENCES officer(id)
   - is_completed BOOLEAN NOT NULL DEFAULT FALSE
   - meta JSONB NOT NULL DEFAULT '{}'
   - INDEX on (session_id, faction_id, is_completed)

Done when: Migration SQL is valid and creates all 4 tables.
