# Plan 11-05: Frontend Faction Politics UI

## Objective
Create the frontend political system UI: faction-specific politics panels showing governance structure, coup/election status, Fezzan loans/intel, and political actions per faction type.

## Context
- @frontend/src/types/tactical.ts -- type definition pattern
- @frontend/src/stores/tacticalStore.ts -- Zustand store pattern
- @frontend/src/components/tactical/BattleMap.tsx -- component pattern reference
- @frontend/src/app/(game)/tactical/page.tsx -- game page route pattern
- @frontend/src/lib/tacticalApi.ts -- API client pattern

## Tasks

### Task 1: Create politics types, API client, and Zustand store
type="auto"

Create:
1. `frontend/src/types/politics.ts`:
   - NobilityRank type (COMMONER | BARON | COUNT | MARQUIS | DUKE)
   - CoupPhase type (PLANNING | ACTIVE | SUCCESS | FAILED | ABORTED)
   - CoupStatus interface -- coupId, phase, leaderId, leaderName, supporterCount, politicalPower, threshold, startedAt
   - CouncilSeat interface -- seatCode, nameKo, officerId?, officerName?, electedAt?, termEndAt?
   - CouncilStatus interface -- seats: CouncilSeat[]
   - Election interface -- electionId, type, startedAt, candidates: Candidate[], isCompleted, winnerId?
   - Candidate interface -- officerId, officerName, votes
   - Loan interface -- loanId, principal, interestRate, remainingDebt, issuedAt, dueAt, isDefaulted
   - IntelligenceOffer interface -- type, nameKo, cost, description
   - FactionPoliticsOverview interface -- factionType, governanceType, leadership, councilStatus?, activeCoup?, activeElection?, loans?, fezzanOperational?

2. `frontend/src/lib/politicsApi.ts`:
   - getPoliticsOverview(sessionId)
   - getCoupStatus(sessionId)
   - initiateCoup(sessionId, factionId)
   - joinCoup(sessionId, coupId)
   - abortCoup(sessionId, coupId)
   - getNobility(sessionId)
   - getCouncilStatus(sessionId)
   - getActiveElection(sessionId)
   - startElection(sessionId)
   - castVote(sessionId, electionId, candidateId)
   - getLoans(sessionId)
   - takeLoan(sessionId, amount)
   - repayLoan(sessionId, loanId, amount)
   - getIntelOffers(sessionId)
   - buyIntel(sessionId, targetFactionId, type)

3. `frontend/src/stores/politicsStore.ts` (Zustand):
   - State: overview, coupStatus, councilStatus, activeElection, loans, intelOffers, loading, error
   - Actions: fetchOverview, fetchCoupStatus, fetchCouncil, fetchElection, fetchLoans, fetchIntel, performCoupAction, performElectionAction, performLoanAction, performIntelAction

4. Export from `frontend/src/types/index.ts`

Done when: All types, API functions, and store compile without TypeScript errors.

### Task 2: Create faction-specific politics UI components and page
type="auto"

Create:
1. `frontend/src/components/politics/EmpirePanel.tsx`:
   - Shows current Emperor/Sovereign name and portrait
   - Nobility hierarchy list (officers grouped by peerage)
   - Coup section: if active coup, show phase/power/threshold progress bar
   - Actions: "Initiate Coup" button (rank >= 5), "Join Coup" button, "Abort Coup" button
   - Conditional rendering based on coup phase

2. `frontend/src/components/politics/AlliancePanel.tsx`:
   - Supreme Council table: 11 seats with holder name, elected date, term remaining
   - Current election: candidate list with vote counts, "Cast Vote" button
   - Start election button (for council chair only)
   - Confidence vote section

3. `frontend/src/components/politics/FezzanPanel.tsx`:
   - Active loans table: principal, interest rate, remaining debt, due date, status badge (active/overdue/defaulted)
   - "Take Loan" form: amount input with max calculation
   - "Repay Loan" form: select loan, amount input
   - Intelligence market: cards showing available intel types with cost and "Buy" button
   - Total debt summary with warning colors (green < 50%, yellow 50-80%, red > 80% of max)

4. `frontend/src/components/politics/PoliticsOverview.tsx`:
   - Renders the correct panel based on factionType (empire -> EmpirePanel, alliance -> AlliancePanel)
   - Always shows FezzanPanel (loans available to both factions)
   - Header showing governance type and current leader
   - Loading/error states

5. `frontend/src/app/(game)/politics/page.tsx`:
   - Game page route for /politics
   - Fetches overview on mount
   - Renders PoliticsOverview component

Done when: All components render without errors. Page route is accessible. Faction-specific panels show correct content.
