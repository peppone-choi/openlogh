# Plan 11-03: Fezzan NPC - Trade, Loans & Intelligence Market

## Objective
Implement Fezzan as an NPC faction: loan system with interest/default mechanics, intelligence trading, trade route bonuses, and the Fezzan ending (debt domination).

## Context
- @backend/game-app/src/main/kotlin/com/openlogh/entity/Faction.kt -- faction entity with factionType field
- @backend/game-app/src/main/kotlin/com/openlogh/engine/DiplomacyService.kt -- inter-faction relations
- @backend/game-app/src/main/kotlin/com/openlogh/engine/EconomyService.kt -- economic calculations
- @docs/reference/scenarios_detail.md -- "Fezzan debt default = Fezzan ending", loan mechanics
- V38 migration from Plan 11-01 creates fezzan_loan table

## Tasks

### Task 1: Create Fezzan loan and trade entities/services
type="auto"

Create:
1. `FezzanLoan` JPA entity in `com.openlogh.entity`:
   - Maps to fezzan_loan table from V38
   - id, sessionId, borrowerFactionId, principal, interestRate, remainingDebt, issuedAt, dueAt, repaidAt, isDefaulted, meta

2. `FezzanLoanRepository` in `com.openlogh.repository`

3. `IntelligenceType` enum in `com.openlogh.model`:
   - FLEET_POSITIONS (함대 위치 -- reveals enemy fleet locations)
   - PLANET_RESOURCES (행성 자원 -- reveals enemy planet stats)
   - OFFICER_INFO (장교 정보 -- reveals enemy officer stats)
   - MILITARY_POWER (군사력 -- reveals enemy faction military power)
   - COUP_INTEL (쿠데타 정보 -- reveals ongoing coup if any)
   Each has: code, nameKo, cost (Int -- funds required), duration (Int -- how many ticks the intel lasts)

4. `FezzanService` in `com.openlogh.service`:
   **Loan system:**
   - `offerLoan(sessionId, borrowerFactionId, amount)` -- Fezzan offers loan
     - Interest rate: 5% base + 2% per existing unpaid loan
     - Max total debt: faction's total funds * 3
     - Due date: 30 game-days (30 * 24 * 60 * 60 / 24 ticks)
   - `repayLoan(sessionId, loanId, amount)` -- partial or full repayment
     - Deducts from borrower faction funds
     - If remainingDebt <= 0: mark repaid
   - `processLoanTick(sessionId)` -- check all active loans
     - Apply daily interest (interestRate / 30 per tick-day)
     - Check due dates: if past due and not repaid -> isDefaulted = true
     - If defaulted: apply penalty (increase tax_rate, decrease approval on all planets)
   - `checkFezzanEnding(sessionId)` -- if ANY faction has 3+ defaulted loans -> Fezzan ending
     - Returns FezzanEndingResult (triggered: Boolean, dominatedFactionId: Long?)
   - `getTotalDebt(sessionId, factionId)` -- sum of remainingDebt for faction
   - `getActiveLoans(sessionId, factionId)` -- list non-repaid loans

   **Intelligence market:**
   - `purchaseIntelligence(sessionId, buyerFactionId, targetFactionId, type: IntelligenceType)` -- buy intel from Fezzan
     - Deducts cost from buyer's funds
     - Fezzan must be alive (not absorbed into Empire in later scenarios)
     - Returns intel data based on type
   - `isFezzanOperational(sessionId)` -- check if Fezzan faction still exists and is neutral

   **Trade route bonuses:**
   - `calculateTradeBonus(sessionId, factionId)` -- planets with trade_route connected to Fezzan get +20% commerce
   - Called by EconomyService during economic processing

Done when: FezzanLoan entity compiles. FezzanService has loan, intelligence, and trade methods. Fezzan ending check implemented.

### Task 2: Create Fezzan AI tick processor
type="auto"

Create `FezzanAiService` in `com.openlogh.engine`:
1. `processFezzanTick(sessionId)` -- main AI loop called each tick:
   - If Fezzan is not operational (absorbed): skip
   - Every 100 ticks: evaluate loan offers to factions
     - Offer loans to factions with funds < 50% of average
     - Loan amount: 20-40% of Fezzan's own funds
   - Every 50 ticks: adjust intelligence prices based on war state
     - During active wars: prices +50%
     - During peace: prices normal
   - Track Fezzan's own funds: gains from interest payments, intelligence sales
   - Fezzan neutrality: never attacks, never allies
   - If Fezzan has < 10% of starting funds: become desperate (lower loan rates to attract borrowers)

2. `FezzanEndingService` in `com.openlogh.service`:
   - `triggerFezzanEnding(sessionId, dominatedFactionId)` -- execute Fezzan ending
     - Set dominated faction's sovereignty to Fezzan control
     - Broadcast event: "페잔 자치령이 [진영]을 경제적으로 지배하게 되었다!"
     - End the game session with result "fezzan_domination"
   - Called from FezzanService.checkFezzanEnding when condition met

Done when: FezzanAiService compiles. AI evaluates loan offers and intelligence pricing. Fezzan ending triggers correctly.
