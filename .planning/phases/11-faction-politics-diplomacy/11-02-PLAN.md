# Plan 11-02: Alliance Politics - Supreme Council & Elections

## Objective
Implement the Alliance's democratic governance: Supreme Council with 11 seats, periodic election system, and council vote mechanics for policy decisions.

## Context
- @backend/game-app/src/main/kotlin/com/openlogh/entity/Faction.kt -- faction entity
- @backend/game-app/src/main/kotlin/com/openlogh/model/PositionCard.kt -- SUPREME_COUNCIL_CHAIR, COUNCIL_DEFENSE_CHAIR etc.
- @backend/game-app/src/main/kotlin/com/openlogh/service/PersonnelService.kt -- rank/appointment logic
- @docs/reference/scenarios_detail.md -- Greenhill coup (political machination threshold)
- V38 migration from Plan 11-01 creates council_seat and election tables

## Tasks

### Task 1: Create Alliance political model and services
type="auto"

Create:
1. `CouncilSeatCode` enum in `com.openlogh.model` -- 11 Supreme Council seats:
   - SUPREME_COUNCIL_CHAIR (최고평의회 의장)
   - DEFENSE_COMMITTEE_CHAIR (국방위원장)
   - FINANCE_COMMITTEE_CHAIR (재정위원장)
   - FOREIGN_AFFAIRS_CHAIR (외교위원장)
   - INTELLIGENCE_CHAIR (정보위원장)
   - LOGISTICS_CHAIR (군수위원장)
   - PERSONNEL_CHAIR (인사위원장)
   - JUSTICE_CHAIR (법무위원장)
   - INTERIOR_CHAIR (내무위원장)
   - TECHNOLOGY_CHAIR (기술위원장)
   - ECONOMIC_CHAIR (경제위원장)
   Each has: code, nameKo, nameEn, relatedPositionCard (PositionCard?), minRank (Int)

2. `ElectionType` enum in `com.openlogh.model`:
   - COUNCIL_CHAIR (의장 선거 -- full council re-election)
   - SINGLE_SEAT (보궐 선거 -- fill a single vacancy)
   - CONFIDENCE_VOTE (신임투표 -- vote of confidence/no-confidence)

3. `CouncilSeat` JPA entity in `com.openlogh.entity`:
   - Maps to council_seat table from V38
   - id, sessionId, factionId, seatCode, officerId, electedAt, termEndAt, votesReceived, meta

4. `CouncilSeatRepository` in `com.openlogh.repository`

5. `Election` JPA entity in `com.openlogh.entity`:
   - Maps to election table from V38
   - id, sessionId, factionId, electionType, startedAt, endedAt, candidates (JSONB), votes (JSONB), winnerOfficerId, isCompleted, meta

6. `ElectionRepository` in `com.openlogh.repository`

7. `AlliancePoliticsService` in `com.openlogh.service`:
   - `initializeCouncil(sessionId, factionId)` -- create 11 empty seats
   - `nominateCandidate(sessionId, factionId, seatCode, officerId)` -- add candidate to election
   - `castVote(sessionId, electionId, voterId, candidateId)` -- record a vote
   - `resolveElection(sessionId, electionId)` -- tally votes, seat winner, assign position card
   - `startElection(sessionId, factionId, type, seatCode?)` -- create Election entity
   - `callConfidenceVote(sessionId, factionId, targetOfficerId)` -- initiate no-confidence vote
   - `getCouncilStatus(sessionId, factionId)` -- return all seats with holders
   - `processElectionTick(sessionId)` -- check for expired elections, auto-resolve

Done when: All entities, enums, and service compile. Council has 11 seat types. Elections support 3 types.

### Task 2: Create Empire politics service (CoupService)
type="auto"

Create `EmpirePoliticsService` in `com.openlogh.service`:
1. `initiateCoup(sessionId, factionId, leaderId)` -- create CoupEvent in PLANNING phase
   - Validates leader rank >= 5 (Commodore+)
   - Validates faction is Empire type
2. `joinCoup(sessionId, coupId, officerId)` -- add supporter
   - Cannot join if rank >= 9 (the sovereign)
3. `addPoliticalPower(sessionId, coupId, points)` -- accumulate machination points
4. `checkCoupTrigger(sessionId, coupId)` -- if politicalPower >= threshold, transition PLANNING->ACTIVE
5. `resolveCoup(sessionId, coupId, success: Boolean)` -- transition to SUCCESS/FAILED
   - SUCCESS: overthrow sovereign, leader becomes new sovereign, supporters get nobility upgrades
   - FAILED: leader and supporters get demoted, arrested (blockState = 1)
6. `abortCoup(sessionId, coupId)` -- leader can abort during PLANNING
7. `getCoupStatus(sessionId, factionId)` -- return active coup if any
8. `processCoupTick(sessionId)` -- check active coups, resolve based on military power comparison
   - Compare total military power of coup supporters vs loyalists
   - If supporters > loyalists * 1.2: SUCCESS
   - If loyalists > supporters * 1.5 after 10 ticks: FAILED
9. `grantNobility(sessionId, officerId, rank: NobilityRank)` -- store in officer.meta
10. `getNobility(officer: Officer): NobilityRank` -- read from officer.meta, default COMMONER

Done when: Service compiles. Coup state machine transitions work. Nobility stored in officer.meta.
