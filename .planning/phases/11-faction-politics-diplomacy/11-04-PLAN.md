# Plan 11-04: Politics REST API, DTOs & Tick Integration

## Objective
Create REST controllers, DTOs, and integrate political systems into the game tick engine so politics processes every tick alongside existing systems.

## Context
- @backend/game-app/src/main/kotlin/com/openlogh/controller/TacticalBattleRestController.kt -- REST pattern reference
- @backend/game-app/src/main/kotlin/com/openlogh/dto/TacticalBattleDtos.kt -- DTO pattern reference
- @backend/game-app/src/main/kotlin/com/openlogh/engine/TickEngine.kt -- tick processing loop
- @backend/game-app/src/main/kotlin/com/openlogh/service/GameEventService.kt -- WebSocket event broadcast

## Tasks

### Task 1: Create DTOs and REST controllers
type="auto"

Create:
1. `FactionPoliticsDtos.kt` in `com.openlogh.dto`:
   - `CoupStatusDto` -- coupId, phase, leaderId, leaderName, supporterCount, politicalPower, threshold, startedAt
   - `NobilityDto` -- officerId, officerName, rank (NobilityRank code), nameKo, politicsBonus
   - `CouncilStatusDto` -- seats: List<CouncilSeatDto>
   - `CouncilSeatDto` -- seatCode, nameKo, officerId?, officerName?, electedAt?, termEndAt?
   - `ElectionDto` -- electionId, type, startedAt, candidates: List<CandidateDto>, isCompleted, winnerId?
   - `CandidateDto` -- officerId, officerName, votes
   - `LoanDto` -- loanId, principal, interestRate, remainingDebt, issuedAt, dueAt, isDefaulted
   - `IntelligenceOfferDto` -- type, nameKo, cost, description
   - `FactionPoliticsOverviewDto` -- factionType, governanceType (autocracy/democracy/npc), leadership, councilStatus?, activeCoup?, activeElection?, loans?, fezzanOperational?
   - Request DTOs: `InitiateCoupRequest(factionId)`, `CastVoteRequest(electionId, candidateId)`, `TakeLoanRequest(amount)`, `RepayLoanRequest(loanId, amount)`, `BuyIntelRequest(targetFactionId, intelligenceType)`

2. `FactionPoliticsController.kt` in `com.openlogh.controller`:
   - `GET /api/world/{sessionId}/politics/overview` -- returns FactionPoliticsOverviewDto for player's faction
   - `GET /api/world/{sessionId}/politics/empire/coup` -- get coup status (Empire only)
   - `POST /api/world/{sessionId}/politics/empire/coup/initiate` -- start coup
   - `POST /api/world/{sessionId}/politics/empire/coup/{coupId}/join` -- join coup
   - `POST /api/world/{sessionId}/politics/empire/coup/{coupId}/abort` -- abort coup
   - `GET /api/world/{sessionId}/politics/empire/nobility` -- list nobility ranks in faction
   - `GET /api/world/{sessionId}/politics/alliance/council` -- get council status
   - `GET /api/world/{sessionId}/politics/alliance/election` -- get active election
   - `POST /api/world/{sessionId}/politics/alliance/election/start` -- start election (council chair only)
   - `POST /api/world/{sessionId}/politics/alliance/election/vote` -- cast vote
   - `GET /api/world/{sessionId}/politics/fezzan/loans` -- get active loans for player's faction
   - `POST /api/world/{sessionId}/politics/fezzan/loan/take` -- take loan from Fezzan
   - `POST /api/world/{sessionId}/politics/fezzan/loan/repay` -- repay loan
   - `GET /api/world/{sessionId}/politics/fezzan/intel` -- list available intelligence
   - `POST /api/world/{sessionId}/politics/fezzan/intel/buy` -- purchase intelligence

Done when: All DTOs and controller endpoints compile. Endpoints match service methods.

### Task 2: Integrate politics into tick engine and event broadcast
type="auto"

Modify:
1. `TickEngine.kt` -- add political processing to the tick loop:
   - After existing processing, call:
     - `empirePoliticsService.processCoupTick(sessionId)` for Empire factions
     - `alliancePoliticsService.processElectionTick(sessionId)` for Alliance factions
     - `fezzanService.processLoanTick(sessionId)` for loan interest/defaults
     - `fezzanAiService.processFezzanTick(sessionId)` for Fezzan AI decisions
   - Check `fezzanService.checkFezzanEnding(sessionId)` every 100 ticks

2. `GameEventService.kt` -- add political event types:
   - `broadcastCoupEvent(sessionId, coupId, phase)` -- notify all empire officers of coup state change
   - `broadcastElectionEvent(sessionId, electionId, type)` -- notify all alliance officers of election
   - `broadcastLoanEvent(sessionId, factionId, loanEvent)` -- notify faction of loan status change
   - `broadcastFezzanEnding(sessionId)` -- game-ending broadcast

3. Wire services into `TickEngine` constructor via Spring DI.

Done when: TickEngine processes politics each tick. Events broadcast via WebSocket. All wiring compiles.
