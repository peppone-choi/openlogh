# Phase 12 Plan 05: Frontend Victory, Chat & Mail UI

## Objective
Build frontend components for victory screen, chat panel, mail UI, and session rankings.

## Context
- @frontend/src/stores/ — existing Zustand stores
- @frontend/src/app/(game)/ — game route pages
- Previous phases use React + Tailwind CSS + Zustand pattern

## Tasks

### Task 1: Victory screen and rankings UI
type="auto"

**Do:**
1. Create `frontend/src/types/victory.ts`:
   - `VictoryResult`, `VictoryTier`, `VictoryCondition`, `SessionRanking` types
2. Create `frontend/src/services/victoryApi.ts`:
   - `getVictoryResult(sessionId)`, `getRankings(sessionId)`, `restartSession(sessionId, scenarioId)`
3. Create `frontend/src/stores/victoryStore.ts` (Zustand):
   - State: `victoryResult`, `rankings`, `isSessionEnded`
   - Actions: `fetchVictoryResult`, `fetchRankings`, `restartSession`
   - WebSocket listener for victory events
4. Create `frontend/src/app/(game)/victory/page.tsx`:
   - Victory banner with condition type and tier (Korean labels)
   - Winner/loser faction display
   - Rankings table: rank, name, faction, score, merit, kills
   - Restart button (admin only) with scenario selector
5. Create `frontend/src/components/game/VictoryBanner.tsx`:
   - Overlay component shown when session ends
   - Links to full victory page

**Done when:**
- Victory page shows result with tier and rankings
- VictoryBanner overlay triggers on session end
- Restart UI works for admins

### Task 2: Chat panel and mail UI
type="auto"

**Do:**
1. Create `frontend/src/types/chat.ts`:
   - `ChatMessage`, `ChatScope`, `MailCount`, `AddressBookEntry` types
2. Create `frontend/src/services/chatApi.ts`:
   - `sendChat(sessionId, message, scope)`, `getChatHistory(sessionId, scope, scopeId)`
   - `getMailCount(sessionId)`, `getAddressBook(sessionId)`, `searchOfficers(sessionId, query)`
3. Create `frontend/src/stores/chatStore.ts` (Zustand):
   - State: `messages` by scope, `currentScope`, `addressBook`
   - Actions: `sendMessage`, `switchScope`, `loadHistory`
   - WebSocket subscription management for chat channels
4. Create `frontend/src/components/game/ChatPanel.tsx`:
   - Scope tabs: Planet / Faction / Global
   - Message list with officer name, timestamp, content
   - Input field with send button, 2-second rate limit indicator
   - Collapsible sidebar panel (resizable)
5. Create `frontend/src/components/game/MailPanel.tsx`:
   - Inbox with message count badge (X/120)
   - Compose form with address book dropdown + search
   - Read/unread status, delete, reply actions
   - Tabs: Private / Faction / Diplomacy
6. Integrate ChatPanel into game layout as persistent sidebar element

**Done when:**
- Chat panel shows messages per scope with real-time updates
- Mail panel shows inbox with compose, reply, delete
- Address book enables easy recipient selection
- Chat integrated into game layout

## Verification
- TypeScript types compile
- Victory page route exists at /victory
- ChatPanel renders with 3 scope tabs
- MailPanel shows inbox with count badge

## Success Criteria
- Victory screen displays when session ends
- Rankings table shows all participants sorted by score
- Chat works in real-time per scope
- Mail UI manages messages within 120 cap
