# Phase 12 Plan 03: Session Lifecycle Management

## Objective
Implement session end sequence with rankings, Hall of Fame recording, and session restart capability.

## Context
- @backend/game-app/src/main/kotlin/com/openlogh/service/RankingService.kt — existing ranking service
- @backend/game-app/src/main/kotlin/com/openlogh/entity/SessionState.kt — session state

## Tasks

### Task 1: Session end sequence and rankings
type="auto"

**Do:**
1. Create `SessionLifecycleService` in `com.openlogh.service` with:
   - `endSession(sessionId, victoryResult)`: freeze game state, calculate final rankings
   - `calculateRankings(sessionId)`: rank all officers by composite score (merit + rank + kills + territory)
   - `recordHallOfFame(sessionId, rankings)`: persist top performers to hall_of_fame records via existing Message/Record system
2. Create `SessionRanking` data class: `officerId`, `officerName`, `factionId`, `rank`, `score`, `meritPoints`, `kills`, `territoryCaptured`
3. Ranking score formula: `meritPoints * 2 + rank * 1000 + kills * 50 + territoryCaptured * 200`
4. Create Flyway V40 adding `session_ranking` table: `id`, `session_id`, `officer_id`, `officer_name`, `faction_id`, `final_rank`, `score`, `merit_points`, `stats` JSONB, `created_at`
5. Create `SessionRankingRepository`

**Done when:**
- Session end freezes game and calculates rankings
- Rankings persisted to session_ranking table
- Hall of Fame records created for top performers

### Task 2: Session restart capability
type="auto"

**Do:**
1. Add `restartSession(sessionId, newScenarioId)` to `SessionLifecycleService`:
   - Validates session is in ENDED state
   - Preserves player accounts (AppUser linkage)
   - Resets all game entities (officers, planets, factions, fleets)
   - Re-initializes from selected scenario via existing ScenarioService
   - Sets session status back to ACTIVE
2. Create REST endpoints in new `SessionLifecycleController`:
   - `POST /api/v1/world/{sessionId}/end` — admin-triggered end
   - `GET /api/v1/world/{sessionId}/rankings` — get final rankings
   - `POST /api/v1/world/{sessionId}/restart` — restart with new scenario
3. Create DTOs: `SessionRankingDto`, `SessionEndDto`, `SessionRestartRequest`

**Done when:**
- Completed session can be restarted with new scenario
- Player accounts preserved across restarts
- REST API for rankings and restart works

## Verification
- SessionLifecycleService compiles
- V40 migration SQL is valid
- REST endpoints exist for end/rankings/restart

## Success Criteria
- Session end produces ranked list of all participants
- Hall of Fame records top performers
- Restart resets game state while keeping player accounts
