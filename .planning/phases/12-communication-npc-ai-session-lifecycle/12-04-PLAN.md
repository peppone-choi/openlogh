# Phase 12 Plan 04: Communication System (Chat & Mail)

## Objective
Implement location-scoped real-time chat and enhanced mail system with 120-message cap and address book.

## Context
- @backend/game-app/src/main/kotlin/com/openlogh/service/MessageService.kt — existing message service with mailbox types
- @backend/game-app/src/main/kotlin/com/openlogh/controller/MessageController.kt — existing message controller

## Tasks

### Task 1: Location-scoped chat system
type="auto"

**Do:**
1. Create `ChatService` in `com.openlogh.service` with:
   - `sendChat(sessionId, officerId, message, scope)`: send message with location scope
   - `getChatHistory(sessionId, scope, scopeId, limit)`: get recent chat messages
   - Chat scopes: `PLANET` (same planet), `FACTION` (same faction), `GLOBAL` (all players)
2. Create `ChatScope` enum: `PLANET`, `FACTION`, `GLOBAL`
3. Create `ChatMessage` entity (or use existing Message entity with new mailboxCode 'chat'):
   - Reuse existing Message table with `mailboxCode = 'chat_planet'`, `'chat_faction'`, `'chat_global'`
   - `srcId` = officer ID, `destId` = planet/faction ID for scoping
4. Create WebSocket chat channels:
   - `/app/chat/{sessionId}/send` — send chat message
   - `/topic/chat/{sessionId}/planet/{planetId}` — planet-scoped chat
   - `/topic/chat/{sessionId}/faction/{factionId}` — faction-scoped chat
   - `/topic/chat/{sessionId}/global` — global chat
5. Create `ChatController` for WebSocket message handling
6. Rate limit: max 1 message per 2 seconds per officer

**Done when:**
- Chat messages are location-scoped (planet/faction/global)
- WebSocket channels for real-time chat work
- Rate limiting prevents spam

### Task 2: Enhanced mail system
type="auto"

**Do:**
1. Update `MessageService` to enforce 120-message cap per mailbox:
   - On send, count existing messages for recipient
   - If at 120, delete oldest unread or oldest read message
   - Add `getMailboxCount(officerId, mailboxType)` method
2. Create `AddressBookService` in `com.openlogh.service`:
   - `getAddressBook(sessionId, officerId)`: returns list of officers the player has communicated with + faction members
   - `searchOfficers(sessionId, query)`: search officers by name for addressing mail
3. Create `MailController` REST endpoints (enhance existing MessageController):
   - `GET /api/v1/world/{sessionId}/mail/count` — mailbox counts by type
   - `GET /api/v1/world/{sessionId}/mail/addressbook` — address book
   - `GET /api/v1/world/{sessionId}/mail/search?q=name` — officer search
4. Create DTOs: `MailCountDto`, `AddressBookEntry`

**Done when:**
- Mailbox enforces 120-message cap with auto-cleanup
- Address book shows previous contacts + faction members
- Officer search enables addressing mail to any officer

## Verification
- ChatService compiles with 3 scopes
- MessageService enforces 120 cap
- WebSocket chat channels defined
- AddressBookService compiles

## Success Criteria
- Players on same planet can chat in real-time
- Faction-wide and global chat channels work
- Mail inbox auto-manages at 120 message limit
- Address book enables easy message addressing
