# Phase 12 Plan 01: NPC AI Personality Enhancement

## Objective
Enhance NPC AI with personality traits (aggressive/defensive/balanced/political) that produce observably different behavior, handle offline player auto-behavior, and make unselected original characters act autonomously.

## Context
- @backend/game-app/src/main/kotlin/com/openlogh/engine/ai/OfficerAI.kt — existing AI decision engine (classifyGeneral uses command/intelligence stats)
- @backend/game-app/src/main/kotlin/com/openlogh/engine/ai/AIContext.kt — AI context data class
- @backend/game-app/src/main/kotlin/com/openlogh/engine/ai/NpcPolicy.kt — NPC policy system
- @backend/game-app/src/main/kotlin/com/openlogh/engine/ai/FactionAI.kt — faction-level AI

## Tasks

### Task 1: Personality enum and Officer entity update
type="auto"

**Do:**
1. Create `PersonalityTrait` enum in `com.openlogh.engine.ai` with values: `AGGRESSIVE`, `DEFENSIVE`, `BALANCED`, `POLITICAL`, `CAUTIOUS`
2. Create Flyway migration V38 adding `personality` VARCHAR(20) column to `officer` table (default 'BALANCED')
3. Update Officer entity to include `personality` field
4. Create `PersonalityWeights` data class mapping each personality to stat weight multipliers:
   - AGGRESSIVE: attack 1.5x, command 1.3x, defense 0.7x
   - DEFENSIVE: defense 1.5x, administration 1.3x, attack 0.7x
   - BALANCED: all 1.0x
   - POLITICAL: politics 1.5x, intelligence 1.3x, command 0.7x
   - CAUTIOUS: defense 1.2x, intelligence 1.2x, attack 0.8x, mobility 0.8x

**Done when:**
- PersonalityTrait enum exists with 5 values
- V38 migration adds personality column
- PersonalityWeights provides stat multipliers per personality

### Task 2: Personality-driven AI decision making
type="auto"

**Do:**
1. Update `classifyGeneral` in OfficerAI to factor in personality weights when comparing stats
2. Add `personalityBias` method to OfficerAI that adjusts action selection probabilities:
   - AGGRESSIVE: higher war action probability, lower peace action probability
   - DEFENSIVE: prefer fortification, training, retreat earlier
   - POLITICAL: prefer diplomacy, proposals, appointments
   - CAUTIOUS: avoid risky actions, prefer scouting before attacking
3. Update `decideWarAction` and `decidePeaceAction` to consult personality bias
4. Update `AIContext` to include personality field
5. Assign personalities to original characters based on their stat distributions in scenario JSON files

**Done when:**
- Different personality types produce measurably different action distributions
- AIContext carries personality information

### Task 3: Offline player auto-behavior and NPC autonomy
type="auto"

**Do:**
1. Create `OfflinePlayerAIService` in `com.openlogh.service` that:
   - Detects offline players (lastAccess > configurable threshold, default 30 min real-time)
   - Copies player's stat weights as AI personality weights
   - Delegates to OfficerAI.decideAndExecute with npcState temporarily set
2. Update tick engine integration point to call OfflinePlayerAIService for inactive players
3. For unselected original characters (npcState >= 2), ensure OfficerAI uses personality from their scenario data
4. Add `lastAccessAt` tracking to Officer entity if not already present (check existing fields)

**Done when:**
- Offline players get AI actions based on their stat distribution
- Unselected originals use personality-weighted AI
- AI takeover is seamless and reversible when player returns

## Verification
- PersonalityTrait enum compiles with 5 values
- V38 migration SQL is valid
- OfficerAI personality integration compiles without errors
- OfflinePlayerAIService handles offline detection

## Success Criteria
- NPC officers with AGGRESSIVE personality choose war actions more frequently than DEFENSIVE ones
- Offline player characters continue operating via AI
- Original characters that no player selected act autonomously with personality
