# Plan 07-02: RankLadderService + PersonnelService + Controller + DTOs

## Objective
Implement the rank ladder ordering, auto-promotion/demotion engine, manual personnel management with authority checks, REST API, and DTOs.

## Tasks

### Task 1: RankLadderService
type="auto"

Create service with:
- `buildLadder(sessionId, factionId)`: returns officers ordered by rank tier desc, then merit desc, fame desc, total stats desc
- `getPromotionCandidates(sessionId, factionId, tier)`: top of ladder for each tier below Captain
- `autoPromote(sessionId)`: every 30 game days, promote top candidates where headcount allows
- `checkHeadcount(sessionId, factionId, targetTier)`: returns whether there's room at the target rank

**Done criteria:** RankLadderService.kt compiles, has all 4 methods.

### Task 2: PersonnelService
type="auto"

Create service with:
- `promote(officerId, promoterId, sessionId)`: validates authority (SOVEREIGN/MILITARY_MINISTER/PERSONNEL_CHIEF position cards based on target rank), checks headcount, promotes officer (merit->0, revoke position cards except PERSONAL/CAPTAIN/FIEF)
- `demote(officerId, demoterId, sessionId)`: validates authority, demotes officer (merit->100, revoke cards)
- `getPersonnelInfo(officerId, sessionId)`: returns rank, merit, promotion eligibility
- `getRankLadder(sessionId, factionId)`: returns full ladder view

**Done criteria:** PersonnelService.kt compiles, has all 4 methods with authority checks.

### Task 3: PersonnelDtos
type="auto"

Create DTOs:
- `RankLadderEntryDto`: officerId, name, rankTier, rankTitle, meritPoints, famePoints, totalStats
- `PersonnelInfoDto`: officerId, name, rankTier, rankTitle, meritPoints, evaluationPoints, famePoints, promotionEligible, nextRankTitle
- `PromoteDemoteRequest`: officerId
- `PersonnelActionResponse`: success, message, updatedOfficer (PersonnelInfoDto)

**Done criteria:** PersonnelDtos.kt exists with all DTOs.

### Task 4: PersonnelController
type="auto"

Create REST controller at `/api/world/{sessionId}/personnel`:
- `GET /ladder/{factionId}` - rank ladder for faction
- `GET /info/{officerId}` - personnel info for officer
- `POST /promote` - promote officer (requires auth)
- `POST /demote` - demote officer (requires auth)

**Done criteria:** PersonnelController.kt compiles with all 4 endpoints.

### Task 5: Compilation Verification
type="auto"

Run gradle compileKotlin to verify all changes compile correctly.

**Done criteria:** Build succeeds without errors.

## Verification
- All services, DTOs, controller compile
- Authority checks use PositionCard system
- Headcount enforcement works
- Merit reset on promotion/demotion
