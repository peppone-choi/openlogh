    Create `TacticalAI.kt` as a pure object (no Spring DI). Single entry point:

    **`fun decide(ctx: TacticalAIContext): List<TacticalCommand>`**

    Decision pipeline (in order):
    1. **Retreat check**: If `ThreatAssessor.shouldRetreat(ctx)` is true, return `listOf(TacticalCommand.Retreat(battleId, officerId))`. Early return — nothing else matters.
    2. **No enemies**: If `ctx.enemies.isEmpty()`, apply mission-default behavior (CONQUEST: move to target, DEFENSE: hold position, SWEEP: hold position).
    3. **Energy adjustment** via `decideEnergy(ctx, nearestEnemyDistance)`:
       - Distance > BEAM_RANGE (200): favor BEAM energy (long-range sniping) per D-08
       - Distance <= GUN_RANGE (150): favor GUN energy (close combat) per D-08
       - HP ratio < 0.4: shift to SHIELD+ENGINE (survival mode) per D-08
       - Only emit SetEnergy if allocation differs from current `unit.energy`
    4. **Formation adjustment** via `decideFormation(ctx)`:
       - If unit not in active combat (no enemy within 300 units) and formation != profile.preferredFormation, emit SetFormation per D-09
       - In active combat: AGGRESSIVE keeps WEDGE, DEFENSIVE switches to MIXED if in WEDGE, others keep current
    5. **Mission behavior** via `decideMissionAction(ctx, threats)`:
       - **CONQUEST** (per D-01): `isHighThreat` check on nearest enemy. If high-threat, generate UnitCommand to move around threat toward (anchorX, anchorY). If low-threat, SetAttackTarget on weakest nearby enemy. If no enemies in engagement range, move toward (anchorX, anchorY).
       - **DEFENSE** (per D-02): Calculate distance from anchor. If enemy within `preferredEngagementRange * 1.5` of anchor, intercept (SetAttackTarget + move toward enemy). If no enemies near anchor, return to anchor if distance > 50.
       - **SWEEP** (per D-03): Use `rankThreats` to pick target. Focus-fire vs distribute logic per D-10: if SWEEP and ctx.allies.size > 1, distribute top N threats across allies by checking if ally already targets that enemy. Otherwise focus on #1 threat.
    6. **Target selection** via `decideTarget(ctx, threats)`:
       - CONQUEST: lowest threat score (weakest enemy) per D-01 inversion
       - DEFENSE: nearest enemy to anchor point
       - SWEEP: highest threat score per D-03
       - Personality modifier: AGGRESSIVE picks targets with higher HP (wants a fight), CAUTIOUS picks targets with lower HP (safe kill)

    **Helper: `fun computeEnergyAllocation(distanceToEnemy: Double, hpRatio: Double, personality: PersonalityTrait): EnergyAllocation`**
    Returns one of the predefined allocations (OFFENSIVE, DEFENSIVE, BALANCED, EVASIVE) based on distance + HP + personality bias. Not a continuous slider — picks from the 4 presets.

    **Helper: `fun moveToward(unit: TacticalUnit, targetX: Double, targetY: Double, battleId: Long): TacticalCommand.UnitCommand`**
    Generates UnitCommand with dirX/dirY normalized toward target, speed=1.0.

    **Helper: `fun moveAway(unit: TacticalUnit, threatX: Double, threatY: Double, goalX: Double, goalY: Double, battleId: Long): TacticalCommand.UnitCommand`**
    Generates UnitCommand that arcs away from threat while trending toward goal. Used by CONQUEST bypass.

    Create `TacticalAITest.kt` with JUnit 5 tests covering each behavior item. Use test helper to build TacticalAIContext instances with controlled unit positions, HP, morale, and enemy configurations.
