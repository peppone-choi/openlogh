   ```sql
   CREATE TABLE proposal (
       id BIGSERIAL PRIMARY KEY,
       session_id BIGINT NOT NULL,
       requester_id BIGINT NOT NULL REFERENCES officer(id),
       approver_id BIGINT NOT NULL REFERENCES officer(id),
       action_code VARCHAR(100) NOT NULL,
       args JSONB NOT NULL DEFAULT '{}'::jsonb,
       status VARCHAR(20) NOT NULL DEFAULT 'pending',
       reason TEXT,
       created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
       resolved_at TIMESTAMPTZ,
       CONSTRAINT chk_proposal_status CHECK (status IN ('pending', 'approved', 'rejected', 'expired'))
   );
   CREATE INDEX idx_proposal_session ON proposal(session_id);
   CREATE INDEX idx_proposal_approver_status ON proposal(approver_id, status);
   CREATE INDEX idx_proposal_requester ON proposal(requester_id);
   ```

1. Create `ProposalService` in `com.openlogh.service`:

   Dependencies: ProposalRepository, OfficerRepository, CommandExecutor, CommandService (for createCommandEnv), GameEventService (optional, for WebSocket notification), MessageService (for in-game mail notification).

   Methods:

   a. `submitProposal(requesterId: Long, request: SubmitProposalRequest): ProposalResponse`
      - Validate requester exists and belongs to same session/faction as approver
      - Validate approver exists
      - Validate approver holds a position card that can execute the requested command (use PositionCardRegistry.canExecute with approver's cards)
      - Validate requester does NOT have the card (otherwise they should execute directly)
      - Create Proposal entity with status="pending"
      - Send in-game message notification to approver via MessageService: "[requesterName]이(가) [actionCode] 커맨드를 제안했습니다."
      - Optionally fire WebSocket event to notify approver in real-time
      - Return ProposalResponse

   b. `resolveProposal(approverId: Long, proposalId: Long, request: ResolveProposalRequest): ProposalResponse`
      - Load proposal, verify approverId matches
      - Verify status is "pending"
      - If approved:
        - Execute the command via CommandExecutor using approver's authority (cards) but note in logs that it was a proposal from requester
        - Deduct CP from the REQUESTER (not the approver) -- this is the gin7 rule
        - Set status = "approved", resolvedAt = now()
        - Notify requester: "제안이 승인되었습니다: [actionCode]"
      - If rejected:
        - Set status = "rejected", reason = request.reason, resolvedAt = now()
        - Notify requester: "제안이 거부되었습니다: [actionCode] - [reason]"
      - Return ProposalResponse

   c. `listPendingForApprover(approverId: Long): List<ProposalResponse>`
      - Return all proposals where approverId matches and status = "pending"
      - Include requester name from OfficerRepository

   d. `listMyProposals(requesterId: Long): List<ProposalResponse>`
      - Return all proposals where requesterId matches, ordered by createdAt desc
      - Include approver name from OfficerRepository

   e. `findEligibleApprovers(requesterId: Long, actionCode: String): List<EligibleApproverResponse>`
      - Find the CommandGroup for the action code
      - Find all officers in same faction who hold a position card granting that command group
      - Return list of eligible approvers with their names, ranks, and relevant cards
      - Exclude the requester themselves

   f. `expireOldProposals(sessionId: Long)` (called by tick engine or scheduled)
      - Set status = "expired" for pending proposals older than 24 real-time hours (game time: ~30 game days)

   Helper: `toResponse(proposal: Proposal): ProposalResponse` - maps entity to DTO, fetching officer names.

2. Create `ProposalController` in `com.openlogh.controller`:

   ```kotlin
   @RestController
   @RequestMapping("/api/proposals")
   class ProposalController(
       private val proposalService: ProposalService,
       private val commandService: CommandService, // for ownership verification
   ) {
       @PostMapping("/submit/{generalId}")
       fun submit(
           @PathVariable generalId: Long,
           @RequestBody request: SubmitProposalRequest,
       ): ResponseEntity<ProposalResponse> {
           val response = proposalService.submitProposal(generalId, request)
           return ResponseEntity.ok(response)
       }

       @PostMapping("/resolve/{generalId}/{proposalId}")
       fun resolve(
           @PathVariable generalId: Long,
           @PathVariable proposalId: Long,
           @RequestBody request: ResolveProposalRequest,
       ): ResponseEntity<ProposalResponse> {
           val response = proposalService.resolveProposal(generalId, proposalId, request)
           return ResponseEntity.ok(response)
       }

       @GetMapping("/pending/{generalId}")
       fun pendingForApprover(@PathVariable generalId: Long): ResponseEntity<List<ProposalResponse>> {
           return ResponseEntity.ok(proposalService.listPendingForApprover(generalId))
       }

       @GetMapping("/my/{generalId}")
       fun myProposals(@PathVariable generalId: Long): ResponseEntity<List<ProposalResponse>> {
           return ResponseEntity.ok(proposalService.listMyProposals(generalId))
       }

       @GetMapping("/eligible-approvers/{generalId}")
       fun eligibleApprovers(
           @PathVariable generalId: Long,
           @RequestParam actionCode: String,
       ): ResponseEntity<List<EligibleApproverResponse>> {
           return ResponseEntity.ok(proposalService.findEligibleApprovers(generalId, actionCode))
       }
   }
   ```

   All endpoints use generalId as the acting officer. Authentication/ownership verification should follow the existing pattern from CommandController.
