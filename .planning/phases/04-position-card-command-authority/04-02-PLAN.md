   - Instead of storing `toTurnIndex(env) + postReqTurn` (turn-based),
   - Store `OffsetDateTime.now().plusSeconds(postReqTurn.toLong() * command.getDuration().toLong())` as ISO string
   - If postReqTurn is the number of turns to wait and `getDuration()` returns seconds per command, calculate: cooldownEnd = now + (postReqTurn * some base duration).    - Simplification: use `getPostReqTurn()` as a multiplier for a 300-second (5 min) base period: `OffsetDateTime.now().plusSeconds(postReqTurn.toLong() * 300L)`

   Apply same conversion to `checkNationCooldown()` and `applyNationCooldown()`.

   The `toTurnIndex()` private method can be removed or kept for backward compat.

3. **Backward compatibility** - The cooldown parsing should handle both old (Int) and new (String/ISO) formats gracefully during migration. When reading, if value is a Number, treat as legacy turn index and allow (don't block). If value is a String, parse as ISO timestamp.
