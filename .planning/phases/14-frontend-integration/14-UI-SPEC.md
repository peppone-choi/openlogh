## Spacing Scale

Multiples of 4 only. No exceptions for this phase.

| Token | Value | Usage in Phase 14 |
|-------|-------|-------------------|
| xs | 4px | Status marker (●/○/🤖) offset from unit icon, gold-border inset, drag-handle gap |
| sm | 8px | Stat marker → label spacing in InfoPanel, drop-zone header padding, sonner icon-text gap |
| md | 16px | Side-sheet drawer inner padding, modal section padding, drop-zone vertical rhythm |
| lg | 24px | End-of-battle modal section gaps, sub-fleet drawer header → first bucket gap, operation-overlay legend padding |
| xl | 32px | End-of-battle modal outer padding, F1 overlay edge gutter |
| 2xl | 48px | End-of-battle modal max content width breathing room above CTA |
| 3xl | 64px | Reserved — not used in Phase 14 |

**Touch targets:** Mobile is **out of scope** for tactical UI (CONTEXT.md Out of Scope, REQUIREMENTS.md). The 44px mobile minimum already in `globals.css @media (max-width: 768px)` is preserved by inheritance — no Phase 14 component needs to add new touch overrides.

**Konva-coordinate exception:** BattleMap operates in a 1000×1000 game-coordinate space scaled to viewport (BattleMap.tsx lines 19–20). Spacing tokens apply to **HTML overlay layers** (succession countdown HTML overlay, drawer, modal). Konva-internal pixel distances (CRC stroke width, ghost icon offset, succession ring radius) are specified in the dimension-specific sections below as Konva pixels — they are NOT spacing-scale tokens.

## Typography

The project ships **one** font (`DungGeunMo` 8-bit Korean pixel) declared globally. Phase 14 uses 4 sizes × 2 weights, all Korean-optimized at line-height 1.4 (the existing globals.css base is `line-height: 1.3` at 14px, which is preserved as `Body small`).

| Role | Size | Weight | Line Height | Phase 14 Usage |
|------|------|--------|-------------|----------------|
| Body small | 12px | 400 (regular) | 1.4 | InfoPanel field labels, sonner toast body, drop-zone helper text, CRC tooltip body, fog ghost tick stamp |
| Body | 14px | 400 (regular) | 1.4 | Default body — drawer body, modal table cells, command-panel rows, gating tooltip body |
| Label / Heading | 14px | 600 (semibold) | 1.4 | Drop-zone bucket headers, modal table column headers, "지휘권 하 유닛" badge text, succession countdown number |
| Display | 20px | 600 (semibold) | 1.2 | End-of-battle modal "승리"/"패배" header, succession-event toast title |

**Allowed weights:** `400` (regular) and `600` (semibold) only. No `500`, no `700`, no italic. This matches existing `[data-slot='card-title']` (semibold) and `[data-slot='button']` (semibold) treatments in globals.css.

**Numeric content (countdowns, ship counts, merit numbers):** Use `font-mono` (`SFMono-Regular, Menlo, …` from `--font-mono`) so 0/O and 1/I disambiguate — required for the 30→0 succession countdown (FE-04) and the merit breakdown ("기본 80 + 작전 +40 = 총 120", D-33).

**Body line-height rationale:** 1.4 (instead of the 1.5 default) preserves the existing `body { line-height: 1.3 }` density that the legacy game UI relies on. Pixel-font Korean glyphs render best at 1.3–1.4. Recommended default 1.5 from the template is overridden here with explicit project-fit reasoning.

## Color

**60/30/10 palette** sourced from `frontend/src/app/globals.css` `:root` block. All hex values are pre-existing CSS variables — Phase 14 introduces no new base palette tokens.

| Role | Token / Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `--background` `#080c14` | Tactical map space-bg, drawer canvas, modal backdrop, page chrome |
| Secondary (30%) | `--card` `#0f1520` + `--muted` `#151d2c` + `--accent` `#1a2436` | Side-sheet drawer surface, drop-zone bucket bg, modal panels, command-panel rows, info-panel sections |
| Accent (10%) | `--primary` `#10b981` (game-green) | **Reserved-for list below — must not bleed onto generic UI** |
| Destructive | `--destructive` `#ef4444` | Succession-failure toast, gating tooltip "지휘권 없음" warning icon, end-of-battle "패배" header |

**Accent reserved for** (10% rule — exact list, no other elements):

1. **Primary CTA in end-of-battle modal:** "전략맵으로 돌아가기" button (D-32) — only one CTA per modal.
2. **"내 지휘권 하 유닛" gold-border indicator** on `TacticalUnitIcon` — uses `--game-gold` `#f59e0b`, NOT `--primary`. (Gold instead of green because green is reserved for online-status marker; mixing them on the same icon defeats glanceability.)
3. **Succession-completed "지휘 인수" Konva ring effect** (D-16) — green pulse `#10b981`, 0.6s duration.
4. **Online status marker `●`** on `TacticalUnitIcon` corner (D-35) — green `#10b981`.

**NOT accent (intentionally muted to keep 60/30/10 ratio):**
- Drop-zone hover state → uses `--accent` `#1a2436` (the secondary tier), not green
- Selected unit ring → uses faction color (see below), not accent
- Operation overlay route lines → uses faction color, not accent
