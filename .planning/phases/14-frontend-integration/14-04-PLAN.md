<objective>
Add a new WebSocket channel `/topic/world/{sessionId}/operations` and broadcast OperationEventDto payloads on every status transition, so the galaxy map frontend (14-16) can render the operations overlay without polling.

Purpose: D-31 requires WebSocket sync for operations. Without this channel, the frontend would need to poll `/api/operations` every second — wasteful and laggy. Phase 12 already had a sync channel pattern; this plan reuses that pattern.

Output: `OperationEventDto.kt`, broadcast calls in `OperationPlanService` + `OperationLifecycleService`, JUnit test verifying `convertAndSend` fires.
</objective>

    ```kotlin
    package com.openlogh.dto

    /**
     * WebSocket broadcast payload for operation status transitions.
     * Topic: /topic/world/{sessionId}/operations
     * Consumed by frontend galaxyStore (see Phase 14 plan 14-16).
     */
    data class OperationEventDto(
        val type: String,                       // OPERATION_PLANNED | OPERATION_STARTED | OPERATION_COMPLETED | OPERATION_CANCELLED
        val operationId: Long,
        val sessionId: Long,
        val factionId: Long,                    // who owns this operation
        val objective: String,                  // CONQUEST | DEFENSE | SWEEP
        val targetStarSystemId: Long,
        val participantFleetIds: List<Long>,
        val status: String,                     // PENDING | ACTIVE | COMPLETED | CANCELLED
        val timestamp: Long = System.currentTimeMillis(),
    ) {
        companion object {
            fun fromPlan(plan: com.openlogh.entity.OperationPlan, type: String): OperationEventDto =
                OperationEventDto(
                    type = type,
                    operationId = plan.id,
                    sessionId = plan.sessionId,
                    factionId = plan.factionId,
                    objective = plan.objective.name,
                    targetStarSystemId = plan.targetStarSystemId,
                    participantFleetIds = plan.participantFleetIds,
                    status = plan.status.name,
                )
        }
    }
    ```
