# Phase 24-06 — Grid Capacity Enforcement (E39)

## Manual reference

gin7 manual p32 (Chapter 3, 全銀河図): "한 그리드에는 하나의 진영에 속한 300 유닛
(= 300함/유닛 * 유닛수)까지만 들어갈 수 있다. 이미 그 진영의 유닛으로 가득찬 그리드에는
워프 항행으로 진입할 수 없다."

## Current state

`backend/game-app/src/main/kotlin/com/openlogh/command/gin7/operations/WarpNavigationCommand.kt`
is the warp entrypoint. Repo grep confirms there is no capacity check anywhere —
`constraint/FleetCapacityConstraint.kt` does not exist. `FleetMovementService.kt` (if
present) handles turn-based movement but no grid check.

## Proposed design

1. **New constraint** `command/constraint/GridCapacityConstraint.kt`:
   - Input: `destPlanetMapId` (from arg), faction of general, session state
   - Compute: sum of `officer.ships` across all officers currently at that grid who
     belong to the same faction (this is what gin7 calls "유닛수 * 300")
   - Fail if `factionShipCount + general.ships > 300 * MAX_UNITS_PER_GRID_PER_FACTION`
     (= 300 × constant). The constant defaults to 60 per faction = 18,000 ships, matching
     the manual's Fleet max (60 units * 300 ships = 18,000).
2. **Wire** into `WarpNavigationCommand.fullConditionConstraints` + any other movement
   commands (`IntraSystemNavigationCommand`, `LongRangeMoveCommand`).
3. **Test** with `InMemoryTurnHarness` — place 60 officers at grid A, attempt to warp a
   61st, expect failure.

## Effort

1 session. Small surface area, no schema change.
