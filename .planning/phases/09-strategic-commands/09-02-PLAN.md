    object CrcValidator {
        // CRC formula constants (per research, Claude's discretion)
        const val CRC_BASE_RANGE = 50.0
        const val CRC_RANGE_PER_COMMAND = 3.0
        const val CRC_BASE_EXPANSION = 0.5
        const val CRC_EXPANSION_PER_COMMAND = 0.01  // command/100.0

        /**
         * Check if target unit is within commander's current CRC.
         * D-08: binary in/out check, no buffer zone.
         */
        fun isWithinCrc(commander: TacticalUnit, target: TacticalUnit): Boolean

        /**
         * Check if a command from officerId can reach the target unit.
         * Self-commands (officerId == unit.officerId) ALWAYS bypass CRC (Pitfall 1).
         * Fleet commander commands check fleet commander's CRC.
         * Sub-fleet commander commands check sub-fleet commander's CRC + unit assignment.
         */
        fun isCommandReachable(
            cmd: TacticalCommand,
            targetUnit: TacticalUnit,
            hierarchy: CommandHierarchy,
            allUnits: List<TacticalUnit>,
        ): Boolean

        /**
         * Compute CRC CommandRange for an officer's command stat.
         * Formula (per D-05, research):
         *   maxRange = 50.0 + (command * 3.0)
         *   expansionRate = 0.5 + (command / 100.0)
         */
        fun computeCrcRange(commandStat: Int): CommandRange
    }
    ```

    Key logic for `isCommandReachable`:
    1. If `cmd.officerId == targetUnit.officerId` -> return true (self-command bypass)
    2. Find the commanding officer's TacticalUnit in allUnits
    3. If commanding officer is fleet commander AND targetUnit has no subFleetCommanderId (direct unit): check fleet commander's CRC
    4. If commanding officer is a sub-fleet commander: check that targetUnit.subFleetCommanderId == cmd.officerId AND sub-fleet commander's CRC covers target
    5. If commanding officer is fleet commander but targetUnit has a subFleetCommanderId: fleet commander can still issue fleet-wide orders to all units in their CRC

    Distance calculation: reuse sqrt((dx)^2 + (dy)^2) pattern from TacticalBattleEngine (don't hand-roll -- match existing pattern).

    object OutOfCrcBehavior {
        const val HP_RETREAT_THRESHOLD = 0.3
        const val RETREAT_SPEED_FACTOR = 0.8
        const val MAX_AUTONOMOUS_TICKS = 120  // ~2 minutes at 1 tick/sec
        const val BATTLE_BOUNDS_X = 1000.0

        /**
         * Process behavior for a unit that is outside its commander's CRC.
         * D-06: maintain last order + HP<30% AI retreat.
         *
         * @param unit The out-of-CRC unit
         * @param commanderUnit The commanding officer's TacticalUnit (for move-toward fallback)
         * @param currentTick Current battle tick number
         */
        fun processOutOfCrcUnit(
            unit: TacticalUnit,
            commanderUnit: TacticalUnit?,
            currentTick: Int,
        )
    }
    ```

    Logic:
    1. If `!unit.isAlive` -> return (skip dead units)
    2. Calculate HP ratio: `unit.hp.toDouble() / unit.maxHp.coerceAtLeast(1)`
    3. If HP ratio < 0.3 AND `!unit.isRetreating`:
       - Set `unit.isRetreating = true`
       - Calculate retreat direction: toward posX=0 for ATTACKER, posX=BATTLE_BOUNDS_X for DEFENDER
       - Set velocity: `unit.velX = retreatDirection * BASE_SPEED * RETREAT_SPEED_FACTOR`, `unit.velY = 0.0`
    4. If `currentTick - unit.lastCommandTick > MAX_AUTONOMOUS_TICKS` AND commanderUnit is not null:
       - Fallback: move toward commander position (Pitfall 5 avoidance)
       - Calculate direction vector from unit to commander, normalize, apply BASE_SPEED * 0.5
    5. Else: do nothing (maintain last order -- velocity unchanged)

    Test file: OutOfCrcBehaviorTest.kt with tests for all behavior cases. Create test helper to build TacticalUnit instances with specific posX/posY/hp/maxHp/side values.
