# Phase 9 Plan 01: Operation Planning & Warp Navigation Commands

## Objective
Add gin7's core strategic military commands: operation planning (occupy/defend/sweep) and warp navigation for fleet movement between star systems.

## Context
- @backend/game-app/src/main/kotlin/com/openlogh/command/ — existing command pattern
- @backend/game-app/src/main/kotlin/com/openlogh/model/PositionCardRegistry.kt — command group mapping
- @backend/game-app/src/main/kotlin/com/openlogh/entity/StarRoute.kt — route distances

## Tasks

### Task 1: Implement operation planning and warp navigation commands
type="auto"

**New officer commands:**
1. `작전수립` (Operation Planning) - OPERATIONS group, MCP pool
   - Sets a faction-level strategic goal for a target star system
   - Requires OPERATIONS card, costs 2 MCP
   - Args: targetStarId, operationType (occupy/defend/sweep)
   - Stores operation in faction meta as active_operations list
   - Duration 600s (10 min realtime)

2. `워프항행` (Warp Navigation) - OPERATIONS group, MCP pool
   - Moves officer (and fleet if attached) to an adjacent star system via route
   - Requires ships > 0, valid star route exists
   - Travel duration based on route distance * base time (300s per distance unit)
   - Mobility stat reduces travel time: duration * (100 / (100 + mobility))
   - Costs 1 MCP, morale decreases by 3 per distance unit
   - Updates officer.planetId to destination planet

3. `장거리워프` (Long-range Warp) - OPERATIONS group, MCP pool
   - Multi-hop warp navigation (2-3 systems away)
   - Higher cost (2 MCP), higher morale penalty (5 per hop)
   - Requires engine item for 3-hop range
   - Duration = sum of route distances * 300s * (100 / (100 + mobility))

**New faction command:**
4. `작전지시` (Operation Directive) - COMMAND group, MCP pool
   - Faction-level operation order (only faction leaders)
   - Sets strategic objective visible to all faction members
   - Costs 3 MCP, duration 900s
   - Stores in faction.meta["operations"]

**Done criteria:**
- All 4 commands compile and are registered in CommandRegistry
- Commands are mapped in PositionCardRegistry to correct groups
- ArgSchemas defined for commands requiring arguments
