# PRD: 3D 게임 맵 (3d-map)

> **Feature**: 3d-map
> **Created**: 2026-04-03
> **Reference**: https://www.samnet.kr
> **Status**: Draft

---

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 현재 2D HTML/CSS 맵은 전략 게임의 몰입감과 공간 인지를 제한하며, 경쟁 삼국지 웹게임(samnet.kr) 대비 시각적 열위 |
| **Solution** | Three.js 기반 3D 맵 렌더링으로 지형/도시/병력을 입체적으로 표현, 2D/3D 전환 지원 |
| **UX Effect** | 지형 고저차와 성곽 규모를 직관적으로 인지, 병력 이동을 실시간 애니메이션으로 확인 |
| **Core Value** | 게임 몰입감 대폭 향상 + 전략적 판단에 필요한 공간 정보의 시각적 전달력 극대화 |

## 1. Discovery: 기회 분석

### 1.1 핵심 고객 문제 (JTBD)

| Job | 현재 해결 방식 | 불만/제약 |
|-----|---------------|----------|
| 전략적 지형 파악 | 2D 맵 + 도시 아이콘 | 산/평야 등 지형 구분 불가, 공간감 부재 |
| 병력 이동 추적 | 유닛 마커 + 점선 | 실시간 이동 감각 부족, 몰입감 저하 |

| 국가 영역 파악 | 색상 오버레이 | 경계 모호, 영토 확장 체감 부족 |

### 1.2 Opportunity Solution Tree

```
Outcome: 게임 몰입감 + 전략적 판단력 향상
├── Opportunity: 공간 인지력 강화

│   └── Solution: 카메라 자유 조작 (회전/줌/팬)
├── Opportunity: 시각적 피드백 강화
│   ├── Solution: 3D 성곽 모델 (규모별 차등)
│   ├── Solution: 병력 이동 애니메이션
│   └── Solution: 전투/이벤트 파티클 이펙트
└── Opportunity: 접근성 유지
    ├── Solution: 2D/3D 모드 전환
    └── Solution: 모바일 성능 최적화
```

## 2. Strategy: 가치 제안

### 2.1 Value Proposition (JTBD 6-Part)

**When** 플레이어가 게임 맵을 확인할 때,
**I want to** 지형과 도시, 병력 배치를 한눈에 파악하고 싶다.
**So I can** 최적의 전략적 판단을 빠르게 내릴 수 있다.
**Unlike** 현재 2D 평면 맵은 공간감이 없어 지형 이점을 직관적으로 파악하기 어렵다.
**Our solution** Three.js 3D 맵은 지형 고저차, 성곽 규모, 병력 이동을 입체적으로 보여준다.
**Resulting in** 몰입감 있는 게임 경험과 더 정확한 전략적 의사결정.

### 2.2 Lean Canvas

| 항목 | 내용 |
|------|------|
| **Problem** | 2D 맵의 몰입감 한계, 경쟁 게임 대비 시각적 열위 |
| **Customer Segments** | 웹 삼국지 전략 게임 플레이어 (PC + 모바일) |
| **Unique Value** | 웹 브라우저에서 Three.js 기반 고품질 3D 전략 맵 |
| **Solution** | 3D 지형 + 성곽 + 병력 + 카메라 + 2D 폴백 |
| **Channels** | 기존 게임 내 맵 뷰 교체 (점진적 롤아웃) |
| **Revenue** | 게임 리텐션 향상 → 유저 체류 시간 증가 |
| **Cost** | Three.js 학습, 3D 에셋 제작/구매, 성능 최적화 |
| **Key Metrics** | 맵 페이지 체류시간, 3D 모드 사용률, FPS 평균 |
| **Unfair Advantage** | 기존 게임 데이터/로직 완비, 맵 데이터 구조화 완료 |

## 3. Research: 경쟁사 분석

### 3.1 경쟁 제품 비교

| 항목 | samnet.kr | 오픈삼국 (현재) | 오픈삼국 (목표) |
|------|-----------|----------------|----------------|
| **렌더링** | Three.js WebGL | HTML/CSS 2D | Three.js WebGL |
| **지형** | 12×12 타일, 5종 프로시저럴 | 배경 이미지 + 도로 PNG | 타일 기반 3D 지형 |
| **도시** | GLB 4단계 성곽 | 8단계 아이콘 (16~32px) | GLB 성곽 모델 |
| **병력** | GLB 병사 + 진군 애니메이션 | 유닛 마커 + 점선 | 유닛 모델 + 이동 애니메이션 |
| **카메라** | OrbitControls | 없음 (고정) | OrbitControls + 프리셋 |
| **이펙트** | 파티클 (전투/화염/연기) | CSS 애니메이션 (펄스/회전) | 파티클 시스템 |
| **모바일** | FPS 제한, AA OFF | 반응형 스케일링 | FPS 제한 + LOD + AA OFF |
| **2D 폴백** | 모드 전환 지원 | 기본 모드 | 2D/3D 토글 |

## 8. Risk & Mitigation

| 리스크 | 영향 | 대응 |
|--------|------|------|
| 모바일 성능 부족 | 높음 | LOD, FPS 제한, 자동 2D 폴백 |
| 3D 에셋 제작 비용 | 중간 | 프로시저럴 지형, 심플 폴리곤 성곽으로 시작 |
| 기존 인터랙션 깨짐 | 높음 | 2D 모드 100% 유지, 3D는 추가 레이어 |
| WebGL 미지원 기기 | 낮음 | WebGL 감지 → 자동 2D 폴백 |
| 번들 사이즈 증가 | 중간 | Three.js tree-shaking, 동적 임포트 |
